# IdeaHub Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deliver a self-hostable GitHub project-discovery app with persistent datasets, dataset-aware chat, project cards, Docker deployment, and a reusable Codex skill.

**Architecture:** FastAPI mediates GitHub access and owns SQLite persistence. A React/Vite client presents search, dataset, project, and chat flows. Docker Compose runs both services behind Nginx; the bundled skill makes the workflow reusable.

**Tech Stack:** Python 3.11, FastAPI, SQLAlchemy, pytest, React, TypeScript, Vite, Docker Compose, Nginx.

## Global Constraints

- Search public GitHub repositories only and keep GitHub credentials server-side.
- Save every successful search as an immutable dataset with normalized project records.
- Run the core app without an AI provider; enable a provider only through environment configuration.
- Do not attempt remote deployment without explicit server access.
